What are the differences between Local Storage And Session Storage?

The differences between Local Storage and Session Storage are:

  • Local Storage stores data in browser without any expiration data.On the other hand,Session Storage stores data in browser for a session.

  • Local Storage data is not deleted when the browser is closed ,whereas Session Storage data is deleted when the browser is closed

  • Local Storage data is available for future sessions. On the other hand,Session Storage data is not available for future sessions.

What are the differences between Global Scope and Block Scope?

The differences between Global Scope and Block Scope are:

  • Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.On the other hand,Block Scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else.

  • A global variable has global scope and is accessible from anywhere in the code.Whereas,a block variable has block scope and cannot be accessed from outside the block.The keywords let and const are used to define block variables but variables declared with the var keyword can not have block scope.

How Event Loop works in Javascript?

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.

The Event Loop is a mechanism that allows JavaScript to perform non-blocking operations. It is the secret behind JavaScript’s asynchronous programming and gives us an illusion of being multithreaded even though it is single-threaded.

When an asynchronous task is started, it will start running. Once the task is completed, it will again check for any other tasks that need to be performed.

So it can be said that the Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it.

What are the ways to get undefined value ?

The ways to get undefined value are:

  • A variable that has not been assigned a value will give undefined value.

  • Function with no return will be undefined.

  • The value will be undefined if parameter is not passed.

  • If return has nothing on the right side will return undefined value

  • Property that doesn't exist on an object will give you undefined value

  • Accessing array elements outside of the index range will be undefined

  • Deleting an element inside an array.

  • Set a value directly to undefined.
